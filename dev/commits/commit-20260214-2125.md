# Коммит 20260214-2125

## Что исправлено
- Добавлена обязательная директория `web/public/` и файл-заглушка `web/public/.gitkeep`.
- Обновлена документация `docs/DEPLOY.md` с явным правилом про обязательное наличие `web/public/` для Docker-сборки Next.js.

## Почему падало
- Dockerfile для web-сервиса выполняет шаг `COPY --from=builder /app/public ./public`.
- При отсутствии директории `web/public/` путь `/app/public` не создавался в builder-слое, из-за чего сборка падала с ошибкой `not found`.

## Как проверили
- Проверили наличие директории и файла-заглушки в рабочем дереве.
- Попробовали запустить сборку `docker compose -f infra/compose.dev.yml build web --no-cache`, но в текущем окружении отсутствует команда `docker`, поэтому полноценная проверка сборки недоступна.

## Следующий шаг
- Запустить на машине с установленным Docker команды сборки и запуска:
  - `docker compose -f infra/compose.dev.yml build web --no-cache`
  - `docker compose -f infra/compose.dev.yml up -d`
  - `docker ps`
